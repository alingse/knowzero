# KnowZero 设计愿景

> 从零开始，构建你的学习之书

---

## 核心概念

KnowZero 是一个交互式学习平台，AI 作为导师，通过对话生成个性化学习材料。用户通过自然的提问、探索、追问，构建属于自己的知识网络。

### 学习之书

当学习完成后，用户拥有一份完整的「学习之书」：
- 有机组织的文档网络
- 个人标注和思考
- 实体词之间的知识关联
- 循序渐进的学习路径

---

## 核心交互模式

### 模式一：线性学习

```
新主题 → 生成文档 → 阅读 → AI追问 → 点击追问 → 新文档
         ↑                                            │
         └──────────────────继续学习──────────────────┘
```

### 模式二：网状探索

```
         文档A
        /     \
    实体词1   实体词2
      /          \
   文档B ←─────── 文档C
      \          /
       相互关联 → 文档D
```

### 模式三：深度挖掘

```
基础文档
    ↓ 用户: "详细说说"
增强文档 (追加/重写/扩展)
    ↓ 用户: "再详细点"
深入文档
    ↓ 用户: "实战例子"
实战案例
```

## AI 追问设计

### 追问类型

| 类型 | 图标 | 目的 | 示例 |
|------|------|------|------|
| 基础 | 📘 | 澄清概念 | "useState 是什么？" |
| 深入 | 🧠 | 探索原理 | "依赖数组为什么重要？" |
| 实践 | 💡 | 学以致用 | "如何用这个解决问题？" |

### 智能追问策略

```
基于当前文档 → 分析用户水平 → 推荐追问

考虑因素:
- 用户已学过的内容 (避免重复)
- 文档中的实体词 (深入探索)
- 逻辑递进 (从基础到深入)
- 实际应用 (理论与实践结合)
```

## 文档关系

### 关系类型

```
prerequisites: [文档A, 文档B]  ← 学习本主题前需要掌握的
related: [文档C, 文档D]         ← 与本主题相关的
generated_from: 文档E            ← 从哪个文档的实体词生成的
```

### 学习路径

```
文档A ← 前置 ← 文档B ← 前置 ← 文档C
  │                      ↑
  └── 相关 ───────────────┘

用户视角:
"建议先学习 A，再学习 B，最后学习 C"
"B 和 C 都是相关话题"
```
